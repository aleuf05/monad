# Radio Editorial Gate 0.1

1. **Originating intent** — The Lieutenant observed that Fleet Radio still sounds like someone narrating system events and ordered the complete reshape split across two bots.
2. **Verified starting state** — `radio-console/app.js` directly turns baseline counts, vessel status, captain controls, canon ledger changes, route events, escort mode, fuel and watch events into queued speech. Queue scoring controls priority but does not ask whether anyone has a communicative purpose or audience.
3. **Objective / problem** — Make silence the default and air only purposeful vessel communication grounded in real state.
4. **Scope and exclusions** — Bot 1 owns `toys/radio-console/app.js` and its `web/` mirror. Add editorial metadata/gating/topic memory; suppress baseline, canon, control-runtime, routine route/holding narration; reshape posture, fuel, emergency, escort-order and human-watch lines into addressed communications; permit at most one optional acknowledgment. Exclude voice-provider changes, Gemini calls, rich-audio generation, CSS/layout and evaluation tooling.
5. **Constraints / authority** — Preserve real event grounding, human watch messages, emergencies, channel controls, queue priority, diagnostics and browser fallback. Rich audio remains cached-only. Source/live copies move together. No invented contacts, measurements or outcomes.
6. **Acceptance criteria** — Every aired live entry has `topic`, `intent`, `audience` and `editorialReason`; twenty scout route events yield zero calls; canon/control-runtime/routine holding yield zero calls; accepted posture yields one addressed scout call; duplicate posture is suppressed; critical fuel yields one recommendation; emergency separation can interrupt; one topic emits no more than two utterances; routine topic cooldown is at least 45 seconds; suppressed counts/reasons remain inspectable.
7. **Tests / rollback** — JS syntax, drift check, existing Radio tests if present, live HTTPS source markers, and Bot 2's independent evaluation. Roll back the Bot 1 commit to restore the current event narration path.
8. **Assigned actor** — Bot 1, unclaimed. Execute first; Bot 2 may build fixtures concurrently but records final measurements only against Bot 1's commit.
9. **Evidence and completion state** — Authorized and ready to claim. Expected implementation window: roughly two focused hours.
