# Rich Voice API 0.1

1. **Originating intent** — Continue the low-spend rich voice engine toward a live, same-origin surface.
2. **Verified starting state** — Cache/budget/provider core passed tests; loopback port 4775 was unused; no public voice API route or service existed.
3. **Objective / problem** — Add an HTTP boundary that is useful unconfigured, keeps the Gemini key backend-only and cannot accept arbitrary identity/provider prompts.
4. **Scope and exclusions** — Status/budget/estimate/render/artifact endpoints, four bounded synthetic character projections, loopback service unit and tests. Exclude privileged installation/Caddy execution and UI rich-render controls until commissioning.
5. **Constraints / authority** — Gemini only; 1200-character maximum; configured character IDs only; $0.10/300-second daily defaults; absent key returns unconfigured without spend; port loopback-only.
6. **Acceptance criteria** — Free status/estimate work without key; render is cache/budget controlled; immutable WAV is retrievable; invalid character/length rejected; service targets 127.0.0.1:4775.
7. **Tests / rollback** — Core/server unit tests, py_compile and loopback smoke test. Remove service/route and files to roll back.
8. **Assigned actor** — Codex.
9. **Evidence and completion state** — Verified complete and recorded, pending privileged commissioning. Six unit tests pass; Python compilation and diff checks pass. An unconfigured loopback smoke test returned Gemini provider status, zero spend, $0.10/300-second limits and a free estimate/cache key. No API call occurred.
